# Website Review Aspects

A checklist for systematically reviewing a website across business, UX, design, technical, and data dimensions.

---

## 1. Business Layer (Why it exists)

**Understand:**

- Who is the target audience?
- What problem does it solve?
- Primary user journeys
- Revenue model
- Conversion goals

---

## 2. Information Architecture (How content is organized)

**Map:**

- Sitemap
- Navigation hierarchy
- Menus
- Categories
- Internal linking

---

## 3. User Experience (UX)

**Study flows:**

### First-time visitor

```
Landing Page
  → Signup
  → Onboarding
  → Dashboard
```

### Returning user

```
Login
  → Main Action
  → Logout
```

**Review:**

- User journeys
- Click paths
- Number of steps
- Friction points

---

## 4. Visual Design System

**Understand:**

### Layout

- Grid system
- Container widths
- Breakpoints

### Typography

- Fonts
- Sizes
- Weights
- Line heights

### Colors

- Primary
- Secondary
- Accent
- Background

### Components

- Buttons
- Cards
- Tables
- Forms
- Modals

---

## 5. Frontend Behavior

**Observe:**

### Interactions

- Hover states
- Focus states
- Active states
- Loading states

### Animations

- Page transitions
- Scroll effects
- Micro-interactions

### Responsiveness

Check:

- Mobile
- Tablet
- Desktop

---

## 6. Functional Features

Create a complete feature inventory.

**Example:**

| Feature       | Present |
| ------------- | ------- |
| Login         | Yes     |
| Social Login  | Yes     |
| Search        | Yes     |
| Filters       | Yes     |
| Notifications | Yes     |
| Chat          | No      |

**Also document:**

- Validation rules
- Error handling
- Edge cases

---

## 7. Technical Architecture

**Identify:**

### Frontend

- React
- Next.js
- Vue
- Angular

### Backend

- Node.js
- Django
- Laravel
- Rails

### Infrastructure

- CDN
- Hosting
- Cloud provider

### Third-party services

- Stripe
- Auth0
- Firebase
- Intercom
- Analytics

---

## 8. Data & APIs

**Inspect:**

### Data models

- User
- Product
- Order
- Subscription

### APIs

Identify:

- Endpoints
- Request formats
- Response structures
- Pagination
- Authentication

**Understand:**

- What data powers each screen?
